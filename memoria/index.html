Definición del Proyecto

Resumen (Abstract)

Este proyecto consiste en el diseño e implementación de una infraestructura de servicios IT empresarial completa utilizando contenedores Docker. La arquitectura simula un entorno corporativo real segmentado en redes DMZ y LAN, integrando servicios de autenticación centralizada (LDAP), bases de datos, correo electrónico, gestión de proyectos y un portal de empleados dinámico con Single Sign-On (SSO).

Objetivos

  • Desplegar una arquitectura de red segura mediante segmentación (DMZ/LAN).
  • Centralizar la gestión de usuarios y roles mediante OpenLDAP.
  • Implementar un sistema de acceso único (SSO) para aplicaciones internas.
  • Garantizar la alta disponibilidad y modularidad de los servicios mediante Docker Compose.

Justificación

En el panorama tecnológico actual, la contenerización permite a las empresas desplegar servicios de forma rápida y reproducible. Este proyecto demuestra la viabilidad de gestionar una infraestructura compleja con recursos mínimos, garantizando la seguridad mediante un reverse proxy (Nginx) y control de acceso basado en roles (RBAC).

Desarrollo del Proyecto

Arquitectura de Red

La infraestructura se divide en dos redes virtuales aisladas:

Red Subred Propósito
insrv_net 192.168.5.0/24 DMZ: Servicios públicos (Nginx, DNS, Portal).
insrv_local 192.168.51.0/24 LAN: Servicios críticos (LDAP, BBDD, Mail).

Descripción de Servicios

El stack tecnológico se basa en las siguientes imágenes oficiales:

  • OpenLDAP: Motor de autenticación centralizada.
  • Nginx: Gateway de autenticación y Reverse Proxy con soporte SSL/TLS.
  • Apache (PHP): Portal de gestión dinámico que interactúa con LDAP y MySQL.
  • Docker-Mailserver: Servidor de correo integrado con el directorio activo.
  • Redmine: Gestor de proyectos y tareas empresariales.
  • Monitoring (Prometheus/Grafana): Observabilidad del estado de los contenedores.

Flujo de Funcionamiento y Seguridad

El flujo de acceso se basa en el módulo auth_request de Nginx. Cuando un usuario intenta acceder a un servicio LAN, Nginx delega la validación al Portal de Apache, que verifica la sesión y el rol del usuario en LDAP antes de permitir el paso del tráfico.

Conclusiones

La realización de este proyecto ha permitido profundizar en la orquestación de contenedores y en la integración de protocolos de seguridad como LDAPS y HTTPS. Se ha logrado crear un entorno funcional donde la experiencia de usuario mejora gracias al SSO, sin comprometer la seguridad de los datos críticos almacenados en la LAN.

Como aprendizaje clave, destaca la importancia de una correcta segmentación de red para prevenir ataques laterales y la potencia de Docker para prototipar infraestructuras complejas.
